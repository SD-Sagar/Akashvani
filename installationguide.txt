AKASHVANI INSTALLATION & DEPLOYMENT GUIDE
========================================

LOCAL INSTALLATION
------------------
1. Prerequisites:
   - Node.js (v18+)
   - MongoDB (Local or Atlas)
   - Cloudinary Account

2. Setup:
   - In the root folder, run:
     npm run install-all
   
3. Environment Configuration:
   - Navigate to the /backend folder.
   - Create a .env file with the following:
     PORT=5000
     MONGODB_URI=your_mongodb_connection_string
     JWT_SECRET=your_super_secret_key
     NODE_ENV=development
     CLOUDINARY_CLOUD_NAME=your_name
     CLOUDINARY_API_KEY=your_key
     CLOUDINARY_API_SECRET=your_secret

4. Running the App:
   - From the root folder, run:
     npm run dev
   - Frontend will be at: http://localhost:5173
   - Backend will be at: http://localhost:5000


DEPLOYMENT (RENDER.COM)
-----------------------
1. Preparation:
   - Upload your project to GitHub (ensure .gitignore is present).

2. Create a "Web Service" on Render:
   - Connect your GitHub repository.
   - Name: Akashvani
   - Environment: Node
   - Build Command: npm run build
   - Start Command: npm run start

3. Environment Variables (on Render):
   - Add all the variables from your local .env.
   - IMPORTANT: Set NODE_ENV to "production".
   - IMPORTANT: MONGODB_URI must be an Atlas (cloud) URL.

4. Finalize:
   - Render will build the frontend and the backend will serve it automatically.
   - Your app will be live at the provided .onrender.com URL.


TECHNICAL NOTES
---------------
- Build Logic: The `npm run build` command in the root folder installs all dependencies and builds the Vite frontend into the `backend/frontend/dist` folder.
- Real-time: The app uses Socket.io with WebSocket and Polling fallbacks to ensure stability on free hosting tiers like Render.
- Persistence: Unread counts and read receipts are database-backed and persist across sessions.
